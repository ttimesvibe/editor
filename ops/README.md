# Operations Docs

운영 / 인프라 변경 / 환경 재설계 등 **다단계 절차의 사료** 모음.

`CHANGELOG.md` 가 commit 단위 박제라면, 이 폴더는 다단계 절차의 **진행 로그 / 결정 매트릭스 / 사고 박제** 모음. 개별 commit 으로는 흐름을 못 잡는 작업 (예: lab 환경 신설처럼 10+ 단계) 의 사료.

## 형식 규약

새 ops 문서 추가 시:
- **파일명**: `<주제>-<YYYY-MM-DD>.md` (날짜 prefix 가 아닌 suffix — 주제로 먼저 식별)
- **본문 구조**: 배경 → 사용자 결정 매트릭스 → Phase 별 단계 → 진행 로그
- **위 인덱스에 한 줄 추가**
- **CHANGELOG.md 에서도** commit 단위로 본 문서 참조
- **CLAUDE.md 도** 갱신 (현 운영 상태 박제 — 미래 세션이 즉시 발견)

## 디시플린

ops 문서 작성 의무는 다음 중 하나라도 해당하면:
- 다단계 작업 (3 단계 이상)
- 인프라 변경 (worker / KV / repo / DNS / secret 등)
- 사고 발생 후 사후 분석
- 환경 재설계 / 마이그레이션
