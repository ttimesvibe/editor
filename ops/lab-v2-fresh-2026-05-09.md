### S1.1 — `_meta/CHARTER_11TABS_EQUAL_v1.md` (218 줄, v1.1)

**작성**: 2026-05-07 (집 PC). 사용자: "누더기 만들지 맙시다" 명시.
**효력**: 본 헌장 박제 이후 모든 저장·동기화 관련 코드 변경의 기준점.

#### 가치 3 + 우선순위 (헌장의 axiom)
1. **무손실** — 사용자 친 입력 안 사라짐 ★ 최우선
2. **비대칭 0** — 11 탭 같은 계약
3. **다른 사용자 작업 보존**
- 우선순위: **1 ≫ 3**. "충돌 막으려고 저장 자체를 막는 설계는 본말전도" (사고 D `7edf8c0` Forward fix 영역)

#### Dirty 의 명시적 정의
- 사용자 입력 → 데이터 변경 → **dirty=예**
- 그 탭의 PUT 성공 → KV 도달 → **dirty=아니오**
- **fetch / 서버 받기 / 초기 load → dirty 안 만듦** (사용자가 한 게 아니므로)

#### 약속 W (Walls of Durability) — 무손실 4 벽
가치 1 의 보장 = 4 벽 모두 살아있는 것. 한 벽 깨지면 가치 1 자체 깨짐.

| 벽 | 정의 | 트리거 |
|---|---|---|
| **W1** 자동 재시도 | PUT transient failure (5xx/네트워크/타임아웃) backoff (1·2·3 초). 충돌(409)/deleted 는 즉시 W2/충돌 escalate | 시스템 |
| **W2** 로컬 박제 | W1 모두 실패 → 클라 localStorage 자동 박제. 충돌(409) 시에도 박제 (내 데이터 안전망) | 시스템 |
| **W3** 사용자 손에 | JSON 파일 다운로드. 시스템·서버·KV 망가져도 사용자 보유. 수동 저장 실패 모달에 다운로드 버튼 | 사용자 + 시스템 |
| **W4** 떠나기 전 인지 | dirty 있는 상태로 닫기/새로고침/URL 이동 시 beforeunload 알림 | 브라우저 |

**구조적 보장**: W1~W4 = engine 단일 책임으로 구현. 호출자 (App.jsx 등) 는 wiring 분산 X — engine 결과 (저장 상태 / 백업 목록 / 다운로드 트리거) 만 받음. **분산 wiring = 한 곳 빠뜨림으로 가치 1 깨질 위험** (이게 사고 D 의 패턴).

**호환 보존**:
- W2 localStorage 키 형식: `te_backup_{type}_{sessionId}_{timestamp}` (변경 X)
- W3 JSON payload schemaVersion: `"2.0"` (변경 X)

#### 저장 모멘트 보장 (a)(b)(c)
PUT 일어나는 모든 모멘트 (자동 throttle fire / 수동 저장 버튼) 에:
- **(a) 정확성**: dirty 인 모든 탭 모든 변경분 빠짐없이 PUT 에 포함. 11 탭 동등.
- **(b) 비겹침**: baseSavedAt mismatch → 충돌 처리.
  - **같은 sub** 자기 자신 충돌: 자동 통합, ConflictModal 미노출
  - **다른 sub** 충돌: 409 + ConflictModal 노출
- **(c) 부분 실패 격리**: 한 탭 conflict/실패가 다른 탭 PUT 막지 않음

**자동/수동 통일**: 자동 throttle fire 와 수동 저장 버튼 = **동일 PUT 경로**. 두 코드 경로일 수 없음.

#### 1조 — 30초 cascading throttle
- dirty 발생 → 30s timer 시작
- 30s 후 fire → 그 시점 dirty 인 모든 탭 PUT
- PUT 후 dirty 잔존 → 30s timer 새로 시작
- dirty 비면 timer 정지
- 새 dirty → 첫 dirty 부터 시작
- **timer 는 사용자 입력으로만 진행**. 탭 이동/폴링/fetch 영향 X

#### 2조 — 탭 이동이 저장 막지 않음
편집자 탭 이동해도 직전 탭 dirty 는 1조 시계 따라 KV 도달. 이동 자체로 저장 멈추거나 시계 초기화 X.

#### 4조 — 같은 탭 동시 수정 conflict 는 운영 약속
**운영 약속**: "한 사용자가 자신의 어느 디바이스에서든, 다른 사용자가 같은 탭에 동시 들어가지 않음"
- 위반 손실 (30s 윈도우 한쪽 분량 / 자기 다중 디바이스) = **시스템 책임 외**
- 충돌 발견 → ConflictModal 사용자 결정
- **페이지 닫힘 30s 손실도 운영 약속 흡수** — 마지막 작업 후 저장 버튼 + 떠나기 = 사용자 약속

#### 5조 — 11 탭 동등
우선순위/차등 0. 모든 저장 경로 (throttle / 저장 버튼 / fetch / applyServer / lastLoadedAt 추적) = 11 탭 동등 dispatch.

#### 약속 X — 탭 진입 freshness
- 탭 진입 시:
  - dirty=아니오 → KV fresh fetch (화면 = 0초 전 KV)
  - dirty=예 → fetch X (React state 가 더 최신)
- **폴링은 freshness 책임 X**. 폴링의 4 가지 책임만:
  1. heartbeat
  2. active-users 갱신 (단계 A)
  3. 토스트 알림 (다른 편집자 X 탭 수정)
  4. /leave 트리거
- 앱 마운트 = "default 탭 진입" 한 형태. default 탭만 fetch. **prefetch 안 함**.

#### 약속 Y — fetch / 서버 받기 / 초기 load 는 dirty 안 만듦
모든 fetch / 서버 적용 코드 경로는 시작 시점~React state 박힘 직후까지 **dirty 차단 명시 메커니즘** (스위치/플래그). 이름·구조 자유 (예: `justLoadedRef`).

#### 금지 조항 (시스템 차원 강제)
**어떤 저장 경로도 dirty=아니오 인 탭의 데이터를 KV 에 PUT 하지 않음.** 4조 운영 약속을 시스템 차원 강제.

## 도출되는 fresh v2 architecture (자라며 누적)

CHARTER 본질에서 emerge 하는 architecture 영역 (S1.1 단계 도출):

### A1. Storage Engine (단일 책임 모듈)
**책임**: 약속 W (W1~W4) 통합 관리.
**계약**: 호출자에게 (저장 상태 / 백업 목록 / 다운로드 트리거) 만 노출.
**금지**: 호출자 측 wiring 분산 (사고 D 패턴).
**구성**:
- W1 retry: backoff (1/2/3 초), 5xx/network/timeout 만, 409/deleted 는 즉시 escalate
- W2 localStorage: 자동 박제, 키 `te_backup_{type}_{sessionId}_{timestamp}` (호환 보존)
- W3 download: JSON, schemaVersion `"2.0"` (호환 보존)
- W4 beforeunload: dirty 있을 때만

### A2. Dirty Tracking (단일 진실)
**책임**: 11 탭 dirty 상태 관리.
**규칙**:
- 사용자 입력 → dirty=예
- PUT 성공 → dirty=아니오
- fetch/load → 절대 dirty 안 만듦 (약속 Y 메커니즘)
**금기**: dirty=아니오 인 탭 PUT 절대 금지 (헌장 §3)

### A3. Throttle Engine (1조 cascading)
**책임**: 30s cascading throttle.
**규칙**:
- debounce 아님 (timer reset X)
- 사용자 입력으로만 진행
- 탭 이동/폴링/fetch 영향 받지 X
- PUT 후 dirty 잔존 → 새 cycle
**의존**: A2 (dirty), A4 (PUT 경로)

### A4. Save Path (자동/수동 통일)
**책임**: PUT 의 단일 경로. 자동 throttle fire 와 수동 저장 버튼이 같은 코드.
**보장**: (a) 정확성 / (b) 비겹침 / (c) 부분 실패 격리
**의존**: A1 (W1 retry), A2 (dirty), A6 (conflict)

### A5. Tab Dispatch (11 탭 동등)
**책임**: 모든 탭 작업의 단일 dispatch table.
**모듈 covered**:
- SAVE (saveDirtyTabsToKV)
- APPLY_SERVER (ConflictModal "서버 받기")
- FETCH (탭 진입)
- LAST_LOADED_AT (자식 컴포넌트 fetch 동기)
- DIRTY_SNAPSHOT (currentSnapshot 11 탭)
**금기**: 부모/자식 카테고리 분리 (헌장 §6 거부)

### A6. Conflict Handler
**책임**: baseSavedAt mismatch 처리.
**규칙**:
